# Oracle to PostgreSQL Migration Guide

## Overview
Your Azeroth-Library project has been successfully converted from Oracle Database to PostgreSQL. This guide explains all the changes made and the next steps.

## Changes Made

### 1. **Backend Dependencies** (package.json)
- **Removed**: `oracledb` (Oracle driver)
- **Added**: `pg` (node-postgres - PostgreSQL driver)

### 2. **Server Configuration** (server/server.js)

#### Connection Pool
**Oracle (Before):**
```javascript
await oracledb.createPool({
    user: 'admindba',
    password: 'password',
    connectionString: 'localhost/XEPDB1'
});
```

**PostgreSQL (After):**
```javascript
const pool = new Pool({
    user: 'postgres',
    password: 'password',
    host: 'localhost',
    port: 5432,
    database: 'azeroth_library'
});
```

#### Parameter Binding
- **Oracle**: Used named parameters (`:paramName`)
- **PostgreSQL**: Uses numbered parameters (`$1`, `$2`, etc.)

Example change:
```javascript
// Oracle
connection.execute('SELECT * FROM members WHERE MEMBEREMAIL = :email', { email })

// PostgreSQL
pool.query('SELECT * FROM members WHERE member_email = $1', [email])
```

#### Query Results
- **Oracle**: Returns `result.rows`
- **PostgreSQL**: Returns `result.rows` (handled the same way)

#### Stored Procedures
- **Oracle**: Called with `BEGIN...END;` PL/SQL blocks
- **PostgreSQL**: Called as functions using `SELECT function_name(...)`

Example:
```javascript
// Oracle
connection.execute('BEGIN addBook(:bookId, :title, ...); END;', {...})

// PostgreSQL
pool.query('SELECT add_book($1, $2, ...)', [bookId, title, ...])
```

### 3. **Database Schema** (Database_Schema_PostgreSQL.sql)

#### Table Drops
- **Oracle**: Used exception handling with EXECUTE IMMEDIATE
- **PostgreSQL**: Uses `DROP TABLE IF EXISTS ... CASCADE`

#### Auto-Increment IDs
- **Oracle**: `NUMBER GENERATED BY DEFAULT AS IDENTITY`
- **PostgreSQL**: `SERIAL` (equivalent)

#### Data Types
| Oracle | PostgreSQL |
|--------|-----------|
| `VARCHAR2(n)` | `VARCHAR(n)` |
| `NUMBER` | `BIGINT` or `INTEGER` |
| `NUMBER(8,2)` | `NUMERIC(8,2)` |
| `DATE` | `DATE` |
| `SYSDATE` | `NOW()` |

#### Procedures → Functions
- **Oracle**: PL/SQL procedures
- **PostgreSQL**: PL/pgSQL functions

Example:
```sql
-- Oracle
CREATE PROCEDURE AddBook(...) AS BEGIN ... END;

-- PostgreSQL
CREATE FUNCTION add_book(...) RETURNS void AS $$ ... $$ LANGUAGE plpgsql;
```

#### Triggers
- **Oracle**: Uses `:NEW` and `:OLD` references
- **PostgreSQL**: Uses `NEW` and `OLD` without colons
- Both support `BEFORE/AFTER` and `FOR EACH ROW`

#### NVL() vs COALESCE()
- **Oracle**: Uses `NVL(value, default)`
- **PostgreSQL**: Uses `COALESCE(value, default)` (same syntax in both)

### 4. **Column Naming Convention**
- **Oracle**: UPPERCASE (MEMBEREMAIL, MEMBERPASSWORD)
- **PostgreSQL**: lowercase_with_underscore (member_email, member_password)

The PostgreSQL schema uses snake_case conventions for better readability.

## Setup Instructions

### 1. Install Dependencies
```bash
npm install
```

### 2. Create PostgreSQL Database
```bash
# Using psql
createdb azeroth_library

# Or using SQL
CREATE DATABASE azeroth_library;
```

### 3. Load the Schema
```bash
# Using psql
psql -U postgres -d azeroth_library -f Database_Schema_PostgreSQL.sql

# Or copy-paste the contents in pgAdmin query editor
```

### 4. Update Connection Credentials
In `server/server.js`, update the pool configuration with your PostgreSQL credentials:
```javascript
const pool = new Pool({
    user: 'your_postgres_username',      // Default: 'postgres'
    password: 'your_password',
    host: 'localhost',
    port: 5432,
    database: 'azeroth_library'
});
```

### 5. Start the Server
```bash
npm start
```

## Common Issues & Solutions

### Issue: "connect ECONNREFUSED 127.0.0.1:5432"
**Solution**: Ensure PostgreSQL is running and the connection settings are correct.

### Issue: "role "postgres" does not exist"
**Solution**: Update the username in the pool configuration to match your PostgreSQL user.

### Issue: "database 'azeroth_library' does not exist"
**Solution**: Run the command to create the database before loading the schema.

### Issue: Column name case mismatch
**Solution**: Ensure you're using lowercase column names in queries (as defined in the PostgreSQL schema).

## Key Differences Reference

| Aspect | Oracle | PostgreSQL |
|--------|--------|-----------|
| Driver | oracledb | pg |
| Connection | Pool manager | Pool from pg |
| Parameter Binding | `:name` | `$1, $2` |
| Auto-increment | GENERATED...IDENTITY | SERIAL |
| Current Date/Time | SYSDATE | NOW() |
| String Type | VARCHAR2 | VARCHAR |
| NULL Handling | NVL() | COALESCE() |
| Functions | PL/SQL | PL/pgSQL |
| Procedure Call | BEGIN...END; | SELECT function_name() |

## Rollback Plan

If you need to revert to Oracle:
1. Keep the original `Database Schema.sql` file for reference
2. Revert `package.json` to use `oracledb`
3. Restore `server/server.js` from version control
